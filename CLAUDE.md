# 로켓그로스 재고 관리 (nn-rocketgrowth_inventory)

## 프로젝트 개요
쿠팡 로켓그로스 입고 작업을 위한 밀크런 재고 관리 프로젝트.
주기적으로 쿠팡 FC(풀필먼트 센터)에 재고를 입고하는 밀크런 작업을 준비하고 서류를 생성한다.

## 폴더 구조

```
nn-rocketgrowth_inventory/
└── YYYYMMDD(밀크런, 작업자, FC이름, 작업일 MMDD)/   ← 밀크런 회차별 폴더
    ├── raw/                                           ← 원본 데이터
    │   ├── inventory_health_sku_info_YYYYMMDDHHMMSS.xlsx  ← 쿠팡 재고 건강 보고서
    │   ├── Document_YYYY-MM-DD.xls                        ← 쿠팡 발주 문서
    │   └── 로켓그로스-재고체크-MMDD.xlsx                  ← 재고 체크 템플릿
    ├── 밀크런_서류-FC이름/                            ← 생성된 물류 서류
    │   ├── 밀크런_물류동봉문서(거래명세서)_FC_YYYYMMDD.pdf
    │   ├── 밀크런_물류부착문서1 (팔레트부착문서)_FC_YYYYMMDD.pdf
    │   ├── 밀크런_물류부착문서2 (팔레트적재리스트)_FC_YYYYMMDD.xlsx
    │   └── 제품 바코드라벨_FC_YYYYMMDD.pdf
    ├── 작업자_밀크런_취합리스트_YYMMDD_FC이름.xlsx    ← 밀크런 취합 리스트
    ├── 쿠팡 재고이동건_YYYY_MM월.xlsx                 ← 월별 재고 이동 기록
    └── generated_excel (숫자).xlsx                    ← 로켓그로스 입고 신청서
```

## FC(풀필먼트 센터) 목록
- **동탄1** - 주요 FC
- **화성2**
- **천안2**
- **옥천3**

## 주요 파일 설명

### 원본 데이터 (raw/)
| 파일 | 설명 |
|------|------|
| `inventory_health_sku_info_*.xlsx` | 쿠팡 셀러툴에서 다운로드한 재고 건강 정보. SKU별 현재 재고, 판매량, 권장 입고량 포함 |
| `Document_*.xls` | 쿠팡 발주/재고 관련 문서 |
| `로켓그로스-재고체크-*.xlsx` | 재고 수량 체크 및 입고 계획 수립용 템플릿 |

### 작업 파일
| 파일 | 설명 |
|------|------|
| `작업자_밀크런_취합리스트_*.xlsx` | 밀크런 대상 SKU 취합 목록. 시트: 품목리스트, SKU정보-new, 로켓그로스 입고, 입고양식->, 품목원가, Sheet1, 작업, 팔레트체크리스트, hiddenSheet |
| `쿠팡 재고이동건_YYYY_MM월.xlsx` | 월별 FC간 재고 이동 이력. 시트명: MMDD(작업자, 루트, FC이름들) |
| `generated_excel (숫자).xlsx` | 쿠팡 로켓그로스 입고 신청서 (시스템 업로드용). 시트: 로켓그로스 입고, 일괄 하고 입고신청 제외 및 제안사항 |

### 물류 서류 (밀크런_서류-FC/)
| 파일 | 설명 |
|------|------|
| `밀크런_물류동봉문서(거래명세서)_*.pdf` | 상품 동봉 거래명세서 |
| `밀크런_물류부착문서1 (팔레트부착문서)_*.pdf` | 팔레트에 부착하는 식별 문서 |
| `밀크런_물류부착문서2 (팔레트적재리스트)_*.xlsx` | 팔레트별 적재 상품 리스트 |
| `제품 바코드라벨_*.pdf` | 상품별 바코드 라벨 |

## 취합리스트 컬럼 구조 (재고이동건)
```
바코드 | 품목명 | 현재수량 | 입고수량 | # | # | #: 입력하면 안되는 칸
```

## generated_excel 컬럼 구조
```
No. | 등록상품명 | 옵션명 | 판매가 | 전시상품 ID | 등록상품 ID | 옵션 ID | 판매 유형 |
25년 상반기 | 26년 상반기 | 25년 12월 | 26년 01월 | 26년 02월 | 최근 14일 | 2주간 판매순위 | 1주간 판매순위 |
판매량 누적순위 | 판매량 순위 | 풀필먼트풀리스트상의 현재 재고(팔) | 기본 로진양 | 권장 입고 가능 재고 |
입고 수량 입력(필수) | 입고수량에 따른 2주간 판매 예상 | 유통기한 입력(해당 월 필수) | 제조(소비)기한(필수) | 제조일시(필수) |
제조연도(필수) | 품목바코드(필수) | 품목 단위명(필수) | 저온여부(필수) | 판매가원가 | 기타 항목 원본링크 | 카테고리명 | 브랜드사명 | 제조사명 | SKU ID | Req exp date | Req man date | Req prod year
```

## 작업 흐름
1. **재고 확인**: `inventory_health_sku_info_*.xlsx`에서 SKU별 재고 건강 상태 확인
2. **입고 계획**: `로켓그로스-재고체크-*.xlsx`에서 FC별 입고 수량 결정
3. **취합 리스트 작성**: `작업자_밀크런_취합리스트_*.xlsx` 업데이트
4. **입고 신청서 생성**: `generated_excel.xlsx` 생성 (쿠팡 시스템 업로드용)
5. **물류 서류 생성**: 팔레트 문서, 거래명세서, 바코드 라벨 생성
6. **재고 이동 기록**: `쿠팡 재고이동건_YYYY_MM월.xlsx`에 이력 기록

## 네이밍 규칙
- 폴더: `YYYYMMDD(밀크런, 작업자명, FC이름, 작업일 MMDD)`
  - 예: `20260330(밀크런, 서현, 동탄1, 작업일 0326)`
- 물류 서류: `문서종류_FC이름_YYYYMMDD.확장자`
  - 예: `밀크런_물류동봉문서(거래명세서)_동탄1_20260330.pdf`
- 취합리스트: `작업자_밀크런_취합리스트_YYMMDD_FC이름.xlsx`
  - 예: `서현_밀크런_취합리스트_260326_동탄1.xlsx`
- 재고이동: `쿠팡 재고이동건_YYYY_MM월.xlsx` (시트명: MMDD(작업자, 루트, FC목록))
